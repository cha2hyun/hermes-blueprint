# GLOBAL — 전 도메인 공통 규칙

모든 Hermes 워커는 작업 시작 전 이 문서를 읽는다. 도메인 규칙(각 도메인의 AGENTS.md)은 이 문서를 전제로 한다.

## 경로 규칙
- 코드는 `/repos/<도메인>/<프로젝트>`에, 산출물(명세·디자인·리포트)은 `/workspace/<도메인>`의 역할 폴더에 쓴다
- 핸드오프·코멘트에는 요약 + 산출물 경로만 적는다. 전문을 붙여넣지 않는다
- `.hermes` 디렉터리는 어떤 워커도 건드리지 않는다

## 커밋 컨벤션
- 메시지 형식: `[도메인/프로젝트] 메시지` (예: `[dev/payment-web] 결제 버튼 비활성화 조건 추가`)
- 프로젝트 구분이 없는 변경은 `[도메인] 메시지` (예: `[infra] ...`, `[ops] ...`)
- 논리 단위로 분할 커밋. 시크릿 패턴(.env, *.key, 토큰 추정 문자열)은 커밋에서 제외하고 보드에 경고

## worktree 네이밍
- 코딩 태스크는 worktree-per-task: `task-<ID>-<요약>` (예: `task-0142-payment-button`)
- worktree 위치는 `/workspace/<도메인>/worktrees/` — 소모품이며 git 추적 제외

# dev 팀 규칙

../GLOBAL.md를 먼저 읽는다.

## 도메인
소프트웨어 기획·설계·구현·검증 (웹에 한정하지 않음). 팀: planner / designer / coder / qa (+ 공용 ops-worker)
프로젝트 배경·목표는 brief.md 참조.

## 작업 규칙 (골격 — 운영하며 채움)
- 코드는 `/repos/dev/<프로젝트>`, 산출물은 이 폴더의 specs/ design/ qa-reports/에 쓴다
- 명세 없이 구현하지 않는다. 명세에 없는 판단이 필요하면 태스크에 질문을 남긴다
- 테스트 미통과 코드는 완료 처리하지 않는다. 완료 = 커밋까지 마친 상태

## 1. 개요

"디자인까지 신경 쓴 구구단 페이지"를 처음부터 새로 제작한다.
단순 표 출력이 아닌, **인터랙티브 + 시각적 완성도** 두 가지를 모두 갖춘 단일 웹 페이지.

- 대상 단: 2단 ~ 9단 (총 8개)
- 배포 형태: 외부 의존성 없는 단일 HTML 파일
- 목표 경험: 처음 열자마자 보기 좋고, 탭/클릭으로 즉시 반응

---

## 2. 기술 스택

| 항목 | 결정 | 이유 |
|------|------|------|
| 배포 형태 | 단일 HTML 파일 (HTML/CSS/JS 인라인) | 배포 환경 미확정 — 파일 하나 열면 즉시 동작해야 함 |
| 외부 라이브러리 | 사용 금지 | CDN 의존 시 오프라인 환경에서 깨짐 |
| JS 프레임워크 | 순수 Vanilla JS (ES6+) | 빌드 툴체인 불필요, 단순한 도메인에 적합 |
| CSS 방법론 | 인라인 `<style>` + CSS 사용자 정의 속성(Custom Properties) | 단일 파일 내 유지보수 편의 |

## 3. 화면 요구사항

### 3-1. 전체 레이아웃 구조

화면은 3개 영역으로 구성된다:

```
┌──────────────────────────────────────────┐
│  헤더: 페이지 제목 "구구단"              │
├──────────────────────────────────────────┤
│  컨트롤 바: 모드 전환 버튼 + 단 선택    │
├──────────────────────────────────────────┤
│  콘텐츠 영역: 구구단 카드 또는 단일 단  │
└──────────────────────────────────────────┘
```

- 헤더는 상단에 고정 또는 최상위 위치 (sticky/fixed 여부는 designer 결정)
- 컨트롤 바와 콘텐츠 영역은 스크롤 없이 기본 뷰포트(1280×800) 내에서 주요 내용이 보여야 함
- 모바일(360px 이상)에서 레이아웃이 깨지지 않아야 함

### 3-2. 표시 모드 — 두 가지

#### 모드 A: 전체 보기 (기본값, 페이지 최초 로드 상태)

- 2단 ~ 9단을 **8개 카드**로 동시에 표시
- 카드 레이아웃: 반응형 그리드 (열 수는 5절 참조)
- 각 카드 내용:
  - 카드 상단: "N단" 제목 (예: "2단")
  - 카드 본문: N×1=N 부터 N×9=결과 까지 9개 행
- 카드 클릭 시 모드 B로 전환되며 해당 단을 선택 상태로 표시 (필수)

#### 모드 B: 단 선택 보기

- 2~9 중 하나의 단을 선택해서 크게 표시
- 선택된 단의 카드가 콘텐츠 영역 중앙에 넓게 표시됨
- 컨트롤 바의 단 버튼 중 선택된 번호가 활성 상태로 강조됨
- "전체 보기" 버튼 또는 동등한 컨트롤로 모드 A로 돌아갈 수 있음

#### 모드 전환 규칙

| 트리거 | 결과 |
|--------|------|
| 페이지 최초 로드 | 모드 A 활성 |
| 단 버튼(2~9) 클릭 | 모드 B 활성 + 해당 단 표시 |
| 모드 A 카드 클릭 | 모드 B 활성 + 클릭한 단 표시 |
| "전체 보기" 버튼 클릭 | 모드 A 활성 |
| 모드 B에서 다른 단 버튼 클릭 | 모드 B 유지, 표시 단만 교체 |

전환 시 스크롤 위치는 초기화해도 무방.
모드 전환 및 단 변경은 DOM 조작으로 즉시 수행 (fetch/XHR 없음, 클라이언트 사이드).

### 3-3. 식 표시 형식

```
N × M = R
```

- 곱셈 기호는 `×` (HTML entity `&times;`) 사용
- N, M, R 숫자는 고정폭(monospace) 계열 폰트로 표시
- 각 식은 별도 행(row)으로 배치

## 4. 인터랙션 요구사항

### 4-1. 컨트롤 바 구성

컨트롤 바는 두 부분으로 구성된다:

1. **모드 전환 컨트롤**: "전체 보기" 버튼 1개
   - 모드 A 상태에서는 비활성(disabled) 또는 현재 모드 표시
   - 모드 B 상태에서는 클릭 가능, 클릭 시 모드 A로 전환

2. **단 선택 버튼 그룹**: 2, 3, 4, 5, 6, 7, 8, 9 총 8개 버튼
   - 클릭 시 모드 B로 전환 + 해당 단 표시
   - 활성 단 버튼은 시각적으로 강조 (색상 변화, 굵은 글씨 등)
   - 모드 A 상태에서는 어떤 단도 활성 상태가 아님

### 4-2. 상태 피드백

| 상태 | 시각적 피드백 |
|------|---------------|
| 버튼 hover | 배경색 변화 또는 그림자 추가 |
| 단 버튼 선택(active) | 배경색 채움 + 텍스트 색상 반전 또는 굵게 |
| 카드 hover (모드 A) | 카드 그림자 증가 또는 테두리 강조 |
| 전체 보기 버튼 (모드 A에서) | 비활성 또는 현재 상태 표시 |

## 5. 반응형 레이아웃 기준

| 뷰포트 너비 | 모드 A 카드 열 수 | 단 버튼 배치 |
|-------------|-------------------|--------------|
| < 480px (소형 모바일) | 2열 | 2행×4열 또는 1행 8버튼 줄바꿈 허용 |
| 480px ~ 767px (태블릿) | 2~3열 | 1행 8버튼 줄바꿈 허용 |
| ≥ 768px (데스크톱) | 4열 | 1행 8버튼 |

- 가로 스크롤은 어떤 뷰포트에서도 발생하지 않아야 함
- 모드 B 콘텐츠는 뷰포트 너비에 맞게 카드 폭이 조정됨 (max-width 권장)

## 6. 비주얼 가이드라인 (최소 요건)

아래는 구현 최소 요건이다. 구체적인 색상값·여백·타이포그래피는 designer가 결정한다.

### 6-1. 색상

- 각 단 카드마다 **서로 다른 구분 색상** 적용 (8가지, 2단 ~ 9단 각각)
  - 예시 방향: 파스텔 또는 밝은 계열 HSL 팔레트
  - 색상이 카드 배경 또는 카드 상단 헤더 배경에 적용됨
- 선택된 단(모드 B) 카드는 일반 모드 A 카드보다 더 강조된 색상 표현
- 배경: 연한 단색 또는 그라데이션 (카드가 배경 위에 떠 있는 형태)

### 6-2. 타이포그래피

- 폰트: 시스템 폰트 스택 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)
- 수식 부분: 고정폭 폰트 (`'Courier New', Courier, monospace`)
- 페이지 제목(h1): 크게, 중앙 정렬
- 단 제목(카드 내): 중간 크기, 굵게

### 6-3. 카드 스타일

- 카드에 둥근 모서리 (border-radius) 적용
- 카드에 가벼운 그림자 (box-shadow) 적용
- hover 시 그림자 강조 또는 약간의 scale-up 효과 (CSS transition 사용)

## 7. 접근성 / 엣지케이스

### 7-1. 최소 접근성 요건

1. **키보드 접근**: 컨트롤 바의 모든 버튼은 Tab 키로 이동 가능, Enter/Space로 활성화 가능
2. **색상만으로 상태 구분 금지**: 선택된 단 버튼에는 `aria-pressed="true"` 속성 부여 + 시각적 텍스트 강조(굵게 또는 밑줄) 병행
3. **이미지 대체 텍스트**: 장식 외 의미 있는 이미지가 있다면 alt 속성 필수 (현 설계에는 이미지 없음)

### 7-2. 엣지케이스

- 단 숫자 계산: 정수 곱셈이므로 부동소수점 오류 없음 (`Math.round` 불필요)
- 빠른 단 전환 시 깜빡임 없이 즉시 반응 (setTimeout 지연 최소화, 불필요한 animation delay 없음)
- 버튼 중복 클릭: 이미 활성인 단 버튼을 다시 클릭해도 상태 변화 없이 안정적으로 처리

# 구구단 페이지 기능 명세

태스크 ID: t_fe0ecf5e

---

## 1. 개요

사용자가 "디자인까지 신경 쓴 구구단 페이지"를 요청했다.
단순 표 출력이 아니라 **인터랙티브 요소 + 시각적 완성도**를 갖춘 단일 웹 페이지를 만든다.
대상 단: 2단 ~ 9단 (총 8개).

---

## 2. 기술 스택

| 항목 | 결정 | 이유 |
|------|------|------|
| 배포 형태 | 단일 HTML 파일 (HTML/CSS/JS 인라인) | 외부 빌드 의존성 없음, 파일 하나로 즉시 열기 가능 |
| 외부 라이브러리 | 사용 금지 | CDN 없이도 동작해야 함 |
| 프레임워크 | 순수 Vanilla JS | 빌드 툴체인 불필요 |

> 단일 HTML 파일로 권장하는 이유: 배포 환경이 정해지지 않은 시점이므로 "파일을 열면 바로 동작"하는 형태가 가장 안전하다.
> CDN이 필요한 외부 라이브러리(Tailwind CDN, Chart.js 등)는 오프라인 환경에서 깨진다.

## 3. 화면 요구사항

### 3-1. 전체 레이아웃

- 페이지 제목: "구구단" (h1 수준, 화면 상단 중앙)
- 단 선택 컨트롤과 결과 표시 영역이 한 화면 안에 보여야 한다
- 스크롤 없이 주요 컨텐츠를 볼 수 있어야 한다 (데스크톱 기준 1280×800 이상)
- 모바일(360px 이상 너비)에서도 레이아웃이 깨지지 않아야 한다

### 3-2. 단 표시 방식 — 두 가지 모드 지원

#### 모드 A: 전체 보기 (기본값)

- 2단부터 9단까지 **모든 단을 동시에 표시**
- 각 단은 카드(card) 형태로 묶어 표시
- 카드 배치: 데스크톱에서 최소 4열 그리드, 모바일에서 1~2열 그리드
- 각 카드 안에 "N단" 제목 + 9개의 식(N×1=N ~ N×9=81) 목록

#### 모드 B: 단 선택 보기

- 상단 탭 또는 버튼 그룹으로 2~9 숫자를 선택
- 선택된 단 하나만 크게 표시 (카드 또는 전체 너비 영역)
- 선택 상태는 시각적으로 구분 가능해야 함 (활성 탭/버튼 색상 변화)

#### 모드 전환

- 전체 보기 ↔ 단 선택 보기 전환 버튼(또는 토글) 제공
- 전환 시 스크롤 위치는 초기화해도 무방

### 3-3. 결과 표시 형식

각 식은 다음 형식으로 표시한다:

```
N × M = R
```

- 곱셈 기호는 `×` (HTML entity `&times;`) 사용
- N, M, R은 가독성을 위해 고정폭 폰트(monospace) 계열로 표시
- 각 식은 별도 행(row)으로 배치

## 4. 인터랙션 요구사항

| 트리거 | 동작 | 결과 |
|--------|------|------|
| 단 버튼/탭 클릭 | 모드 B 활성화 + 해당 단 선택 | 해당 단 결과 즉시 표시 |
| 전체 보기 버튼 클릭 | 모드 A 활성화 | 8개 카드 전부 표시 |
| 페이지 최초 로드 | 모드 A(전체 보기)가 기본 상태 | 2~9단 전체 표시 |
| 단 카드(모드 A) 클릭 | 해당 단을 모드 B로 확대 표시 | (선택 구현: 필수 아님, 추천) |

> 결과 갱신은 DOM 조작으로 즉시 수행(fetch/XHR 없음, 순수 클라이언트 사이드).

## 5. 반응형 레이아웃 기준

| 뷰포트 너비 | 카드 열 수 (모드 A) | 단 버튼 배치 |
|-------------|---------------------|--------------|
| < 480px (모바일) | 1열 또는 2열 | 2행 4열 또는 1행 8열 (가로 스크롤 없이) |
| 480px ~ 767px (태블릿) | 2~3열 | 1행 8열 또는 줄바꿈 허용 |
| ≥ 768px (데스크톱) | 4열 | 1행 8열 |

## 6. 비주얼 요구사항

아래는 **최소 요건**이다. designer가 구체적인 색상/타이포그래피를 결정한다.

- 각 단 카드마다 **구분 가능한 색상** 적용 (8가지 단이 시각적으로 구분됨)
  - 예시: 2단=파랑, 3단=초록, … 8가지 색상 팔레트 정의
- 선택된 단(모드 B) 카드는 비선택 대비 **강조 표시** (테두리 두껍게 or 배경 더 진하게)
- hover 효과: 카드나 버튼에 마우스 올리면 약간의 시각적 피드백(색상 변화 또는 그림자)
- 배경: 단색 또는 연한 그라데이션 (흰색 배경에 카드가 떠 있는 형태 권장)
- 폰트: 시스템 폰트 스택 사용 (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)

## 7. 접근성 / 엣지케이스

### 최소 접근성 요건 (2가지)

1. **키보드 접근**: 단 선택 버튼/탭은 Tab 키로 이동 가능하고 Enter/Space로 활성화 가능해야 함
2. **색상 의존 금지**: 색상만으로 상태를 구분하지 않는다. 선택된 단 버튼에는 `aria-pressed="true"` 또는 시각적 텍스트 표시(예: 밑줄, 굵은 글씨) 병행

### 엣지케이스

- 단 숫자 표시: 항상 정수, JS `Math.round` 불필요 (정수 곱셈이므로 부동소수점 오류 없음)
- 빠른 탭 전환 시 깜빡임 없이 즉시 갱신 (setTimeout/애니메이션 지연 최소화)

# ops 팀 규칙 (도메인 중립 유틸)

../GLOBAL.md를 먼저 읽는다.

## 도메인
git 유틸(분할 커밋·push), worktree 정리 등 시스템 작업. 팀: ops-worker
코드 작성·분석은 하지 않는다.

## 작업 규칙
- 커밋은 GLOBAL.md 컨벤션(`[도메인/프로젝트] 메시지`)으로, diff를 도메인·논리 단위로 분할
- 시크릿 패턴(.env, *.key, credential, token 추정 문자열) 감지 시 해당 파일 제외 + 보드에 경고
- **push 정책: 초기 2주는 커밋+로컬 브랜치까지만. push는 사람이 개방 선언한 뒤부터** (현재: push 금지)
- push 실패·충돌 시 보드에 기록만 한다. 강제 push 금지
- `.hermes` 디렉터리는 절대 건드리지 않는다
